퍼널 분석

'백문이불여일타 SQL 캠프 입문반' 페이지의 GA 데이터를 활용하여 페이지뷰 → 스크롤 → 신청 버튼 클릭까지의 3단계 퍼널 전환율을 분석했습니다. 유입 채널별 전환율 비교도 함께 수행했습니다.

SQL Product Analytics Funnel 자습

분석 목적

퍼널 분석은 사용자가 특정 행동 흐름(퍼널)을 따라 최종 목표까지 도달하는 비율을 파악하는 기법입니다. 어느 단계에서 사용자가 가장 많이 이탈하는지 확인하고, 개선 방향을 도출하는 것이 목표입니다.

퍼널 구조

3단계 사용자 행동 시퀀스를 정의했습니다.

1단계 · 페이지뷰 (page_view)
100%
2단계 · 스크롤 (scroll)
약 72%
3단계 · 신청 버튼 클릭
약 45%

※ 전환율 수치는 예시입니다.

분석 결과물

단계별 전환율과 사용자 수를 집계한 실제 분석 결과입니다.

핵심 쿼리

STEP 1 · 페이지뷰 세션 수

입문반 페이지를 본 고유 세션 수를 집계합니다. user_pseudo_id와 ga_session_id를 조합해 중복을 제거합니다.

-- 1,135 개의 세션 (PV) select count(distinct user_pseudo_id, ga_session_id) as pv from ga where 1=1 and page_title = '백문이불여일타 SQL 캠프 입문반' and event_name = 'page_view'

STEP 2 · PV → 스크롤 전환율

페이지뷰 이후 스크롤이 발생한 세션만 매칭합니다. 시간 순서(view_time <= scroll_time)를 조건으로 걸어 선후 관계를 보장합니다.

with pv as ( select user_pseudo_id, ga_session_id, event_timestamp_kst as view_time from ga where page_title = '백문이불여일타 SQL 캠프 입문반' and event_name = 'page_view' ), scroll as ( select user_pseudo_id, ga_session_id, event_timestamp_kst as scroll_time from ga where page_title = '백문이불여일타 SQL 캠프 입문반' and event_name = 'scroll' ) select count(distinct pv.user_pseudo_id, pv.ga_session_id) as pv, count(distinct s.user_pseudo_id, s.ga_session_id) as scroll_after_pv, round(count(distinct s.user_pseudo_id, s.ga_session_id) / count(distinct pv.user_pseudo_id, pv.ga_session_id) * 100.0, 2) as pv_scroll_rate from pv left join scroll s on pv.user_pseudo_id = s.user_pseudo_id and pv.ga_session_id = s.ga_session_id and pv.view_time <= s.scroll_time

STEP 3 · PV → 스크롤 → 신청 버튼 클릭 전체 전환율

3단계 퍼널 전체를 연결합니다. 각 단계별 세션 수와 전환율(pv_scroll_rate, scroll_click_rate, pv_click_rate)을 한 번에 계산합니다.

with pv as ( select user_pseudo_id, ga_session_id, event_timestamp_kst as pv_at from ga where page_title = '백문이불여일타 SQL 캠프 입문반' and event_name = 'page_view' ), scroll as ( select user_pseudo_id, ga_session_id, event_timestamp_kst as scroll_at from ga where page_title = '백문이불여일타 SQL 캠프 입문반' and event_name = 'scroll' ), click as ( select user_pseudo_id, ga_session_id, event_timestamp_kst as click_at from ga where page_title = '백문이불여일타 SQL 캠프 입문반' and event_name in ('SQL_basic_form_click', 'SQL_basic_1day_form_click', 'SQL_package_form_click') ) select count(distinct p.user_pseudo_id, p.ga_session_id) as pv, count(distinct s.user_pseudo_id, s.ga_session_id) as scroll_after_pv, count(distinct c.user_pseudo_id, c.ga_session_id) as click_after_scroll, round(count(distinct s.user_pseudo_id, s.ga_session_id) / count(distinct p.user_pseudo_id, p.ga_session_id) * 100, 2) as pv_scroll_rate, round(count(distinct c.user_pseudo_id, c.ga_session_id) / count(distinct s.user_pseudo_id, s.ga_session_id) * 100, 2) as scroll_click_rate, round(count(distinct c.user_pseudo_id, c.ga_session_id) / count(distinct p.user_pseudo_id, p.ga_session_id) * 100, 2) as pv_click_rate from pv p left join scroll s on p.user_pseudo_id = s.user_pseudo_id and p.ga_session_id = s.ga_session_id and p.pv_at <= s.scroll_at left join click c on s.user_pseudo_id = c.user_pseudo_id and s.ga_session_id = c.ga_session_id and s.scroll_at <= c.click_at

STEP 4 · 유입 채널별 퍼널 전환율

어떤 채널(source / medium)로 유입된 사용자가 더 높은 전환율을 보이는지 비교합니다.

with pageview as ( select source, medium, user_pseudo_id, ga_session_id, event_timestamp_kst as pv_at from ga where 1=1 and page_title = '백문이불여일타 SQL 캠프 입문반' and event_name = 'page_view' ), scroll as ( select user_pseudo_id, ga_session_id, event_timestamp_kst as scr_at from ga where 1=1 and page_title = '백문이불여일타 SQL 캠프 입문반' and event_name = 'scroll' ), click as ( select user_pseudo_id, ga_session_id, event_timestamp_kst as clk_at from ga where 1=1 and page_title = '백문이불여일타 SQL 캠프 입문반' and event_name = 'click' ) select p.source, p.medium, count(distinct p.user_pseudo_id, p.ga_session_id) as pv, count(distinct s.user_pseudo_id, s.ga_session_id) as scroll_after_pv, count(distinct c.user_pseudo_id, c.ga_session_id) as click_after_scroll, round(count(distinct s.user_pseudo_id, s.ga_session_id) / count(distinct p.user_pseudo_id, p.ga_session_id) * 100, 2) as pv_scroll_rate, round(count(distinct c.user_pseudo_id, c.ga_session_id) / count(distinct s.user_pseudo_id, s.ga_session_id) * 100, 2) as scroll_click_rate, round(count(distinct c.user_pseudo_id, c.ga_session_id) / count(distinct p.user_pseudo_id, p.ga_session_id) * 100, 2) as pv_click_rate from pageview p left join scroll s on p.user_pseudo_id = s.user_pseudo_id and p.ga_session_id = s.ga_session_id and p.pv_at <= s.scr_at left join click c on s.user_pseudo_id = c.user_pseudo_id and s.ga_session_id = c.ga_session_id and s.scr_at <= c.clk_at group by 1,2 order by 1,2

배운 점

퍼널은 단순한 집계가 아니라 순서 조건이 핵심입니다. 사용자가 page_view 없이 click 이벤트가 발생하는 경우를 어떻게 처리할지, 세션을 어떻게 정의할지가 분석의 질을 결정한다는 점을 배웠습니다.

ROW_NUMBER(), CASE WHEN, NULLIF 등 윈도우 함수와 조건 집계를 실습했습니다.
