퍼널 분석

사용자가 페이지를 보고 → 스크롤하고 → 클릭하기까지의 3단계 행동 흐름을 SQL로 구현하여 각 단계별 전환율을 측정하고 이탈 구간을 파악했습니다.

SQL Product Analytics Funnel 자습

분석 목적

퍼널 분석은 사용자가 특정 행동 흐름(퍼널)을 따라 최종 목표까지 도달하는 비율을 파악하는 기법입니다. 어느 단계에서 사용자가 가장 많이 이탈하는지 확인하고, 개선 방향을 도출하는 것이 목표입니다.

퍼널 구조

3단계 사용자 행동 시퀀스를 정의했습니다.

1단계 · 페이지뷰 (page_view)
100%
2단계 · 스크롤 (scroll)
약 72%
3단계 · 클릭 (click)
약 45%

※ 전환율 수치는 예시입니다.

분석 결과물

단계별 전환율과 사용자 수를 집계한 실제 분석 결과입니다.

핵심 쿼리

각 단계를 세션 단위로 집계하고 순서 조건을 적용해 퍼널 전환율을 계산했습니다.

-- 1단계: 각 사용자의 이벤트 시퀀스를 순서대로 정의 WITH events AS ( SELECT user_id, event_type, event_at, ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY event_at) AS seq FROM user_events WHERE event_type IN ('page_view', 'scroll', 'click') ), -- 2단계: 단계별 도달 여부 집계 funnel AS ( SELECT user_id, MAX(CASE WHEN event_type = 'page_view' THEN 1 ELSE 0 END) AS step1, MAX(CASE WHEN event_type = 'scroll' THEN 1 ELSE 0 END) AS step2, MAX(CASE WHEN event_type = 'click' THEN 1 ELSE 0 END) AS step3 FROM events GROUP BY user_id ) -- 3단계: 전환율 계산 SELECT COUNT(*) AS total_users, SUM(step1) AS reach_step1, SUM(step2) AS reach_step2, SUM(step3) AS reach_step3, ROUND(SUM(step2) * 100.0 / NULLIF(SUM(step1), 0), 1) AS cvr_1_to_2, ROUND(SUM(step3) * 100.0 / NULLIF(SUM(step2), 0), 1) AS cvr_2_to_3 FROM funnel;

배운 점

퍼널은 단순한 집계가 아니라 순서 조건이 핵심입니다. 사용자가 page_view 없이 click 이벤트가 발생하는 경우를 어떻게 처리할지, 세션을 어떻게 정의할지가 분석의 질을 결정한다는 점을 배웠습니다.

ROW_NUMBER(), CASE WHEN, NULLIF 등 윈도우 함수와 조건 집계를 실습했습니다.
