리텐션 분석

코호트 기반으로 사용자 재방문 패턴을 분석했습니다. 가입 주차별로 코호트를 나누고, 이후 주차별 잔존율(Retention Rate)을 SQL로 계산했습니다.

SQL Retention Cohort 자습

분석 목적

리텐션 분석은 "사용자가 돌아오는가?"를 측정하는 핵심 프로덕트 지표입니다. 코호트(가입 시점이 같은 사용자 그룹)별로 주차별 재방문율을 측정하면, 특정 시점에 이탈이 집중되는 패턴과 개선이 필요한 구간을 파악할 수 있습니다.

코호트 리텐션 테이블 (예시)

각 행은 가입 주차 코호트, 각 열은 가입 후 경과 주차를 나타냅니다.

코호트 Week 0 Week 1 Week 2 Week 3 Week 4
2024-W01 100% 68% 51% 44% 38%
2024-W02 100% 71% 55% 47% 40%
2024-W03 100% 65% 49% 41% —

※ 수치는 예시입니다.

핵심 쿼리

-- 1단계: 사용자별 첫 방문 주차(코호트) 계산 WITH cohort AS ( SELECT user_id, DATE_TRUNC('week', MIN(visited_at)) AS cohort_week FROM user_visits GROUP BY user_id ), -- 2단계: 방문 이력과 코호트 정보를 조인 activity AS ( SELECT c.user_id, c.cohort_week, DATE_TRUNC('week', v.visited_at) AS visit_week, DATEDIFF('week', c.cohort_week, DATE_TRUNC('week', v.visited_at)) AS weeks_since_join FROM cohort c JOIN user_visits v USING (user_id) ) -- 3단계: 코호트 × 주차별 잔존율 계산 SELECT cohort_week, weeks_since_join, COUNT(DISTINCT user_id) AS retained_users, ROUND( COUNT(DISTINCT user_id) * 100.0 / FIRST_VALUE(COUNT(DISTINCT user_id)) OVER ( PARTITION BY cohort_week ORDER BY weeks_since_join ), 1 ) AS retention_rate FROM activity GROUP BY cohort_week, weeks_since_join ORDER BY cohort_week, weeks_since_join;

배운 점

코호트를 나누는 기준(일/주/월)과 재방문의 정의(어떤 이벤트를 "방문"으로 볼 것인가)가 결과에 큰 영향을 준다는 것을 알게 됐습니다.

DATE_TRUNC, DATEDIFF, FIRST_VALUE 등 날짜 함수와 윈도우 함수를 활용해 코호트 피벗 테이블을 구성하는 방법을 실습했습니다.
