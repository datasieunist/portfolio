리텐션 분석

코호트 기반으로 사용자 재방문 패턴을 분석했습니다. 가입 주차별로 코호트를 나누고, 이후 주차별 잔존율(Retention Rate)을 SQL로 계산했습니다.

SQL Retention Cohort 자습

분석 목적

리텐션 분석은 "사용자가 돌아오는가?"를 측정하는 핵심 프로덕트 지표입니다. 코호트(가입 시점이 같은 사용자 그룹)별로 주차별 재방문율을 측정하면, 특정 시점에 이탈이 집중되는 패턴과 개선이 필요한 구간을 파악할 수 있습니다.

코호트 리텐션 테이블 (예시)

각 행은 가입 주차 코호트, 각 열은 가입 후 경과 주차를 나타냅니다.

핵심 쿼리

클래식 리텐션은 특정 월에 첫 구매한 고객이 이후 각 월에도 정확히 구매했는지를 측정합니다. customer_stats에서 코호트(첫 구매월)를 정의하고, records와 조인해 월별 재구매 고객 수를 피벗합니다.

with base as ( select c.customer_id , date_format(c.first_order_date, '%Y-%m-01') as first_order_month , date_format(c.last_order_date, '%Y-%m-01') as last_order_month , date_format(r.order_date, '%Y-%m-01') as order_month from customer_stats c join records r on c.customer_id = r.customer_id ) select first_order_month , count(distinct customer_id) as month0 , count(distinct case when order_month = date_add(first_order_month, interval 1 month) then customer_id end) as month1 , count(distinct case when order_month = date_add(first_order_month, interval 2 month) then customer_id end) as month2 , count(distinct case when order_month = date_add(first_order_month, interval 3 month) then customer_id end) as month3 , count(distinct case when order_month = date_add(first_order_month, interval 4 month) then customer_id end) as month4 , count(distinct case when order_month = date_add(first_order_month, interval 5 month) then customer_id end) as month5 , count(distinct case when order_month = date_add(first_order_month, interval 6 month) then customer_id end) as month6 , count(distinct case when order_month = date_add(first_order_month, interval 7 month) then customer_id end) as month7 , count(distinct case when order_month = date_add(first_order_month, interval 8 month) then customer_id end) as month8 , count(distinct case when order_month = date_add(first_order_month, interval 9 month) then customer_id end) as month9 , count(distinct case when order_month = date_add(first_order_month, interval 10 month) then customer_id end) as month10 , count(distinct case when order_month = date_add(first_order_month, interval 11 month) then customer_id end) as month11 from base group by 1 order by 1

배운 점

코호트를 나누는 기준(일/주/월)과 재방문의 정의(어떤 이벤트를 "방문"으로 볼 것인가)가 결과에 큰 영향을 준다는 것을 알게 됐습니다.

DATE_TRUNC, DATEDIFF, FIRST_VALUE 등 날짜 함수와 윈도우 함수를 활용해 코호트 피벗 테이블을 구성하는 방법을 실습했습니다.
